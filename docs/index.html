authenticate

Authenticates a user with their mojang credentials.

authenticate(username: String, password: String, clientToken: String, agent: Object, requestUser: Boolean): Promise<Object>
Parameters
username (String) required. Valid username of a Mojang account
password (String) required. Password for the given account
clientToken (String) optional. Client token for the request
agent (Object) optional. A agent object to get some extra information in the response
Name Description
agent.name String The name of the agent. Can be "Minecraft" or "Scrolls"
agent.version Number Version number of the agent. Use 1 as a default value
requestUser (Boolean) optional. True adds the user object to the response
Returns
Promise<Object>: Promise which resolves to user profile information, clientToken and accessToken

invalidate

Invalidate an obtained access token

invalidate(accessToken: String, clientToken: String): Promise<String>
Parameters
accessToken (String) required. access token to be invalidated
clientToken (String) optional. client token (needs to be identical to the one you used to obtain the access token)
Returns
Promise<String>: resolves if tokens are both valid and invalid

profile

A special service that returns a profile's name, skin, cape, and maybe more.

profile(profileId: String): Promise<Object>
Parameters
profileId (String) required. UUID of a player
Returns
Promise<Object>: Promise which resolves to {id, name, properties}

refresh

Refreshes a given access token

refresh(accessToken: String, clientToken: String, selectedProfile: Object, requestUser: Boolean): Promise<Object>
Parameters
accessToken (String) required. valid access token
clientToken (String) required. client token (needs to be identical to the one used to obtain the access token)
selectedProfile (Object) optional. Used only for access tokens without a selected profile
Name Description
selectedProfile.id String Player id
selectedProfile.name String Player name
requestUser (Boolean) optional. true adds the user object to the response
Returns
Promise<Object>: Promise which resolves to a user object containing a new access token, player and account information

username

Gets the username at a given timestamp

username(name: String, date: Number)
Parameters
name (String) required. The current name of the user
date (Number) optional. UNIX Timestamp to check the username at this time. If null the current time is used. You can use 0 to get the initial name of the account, however this will only work if the name was changed at least once

validate

Checks if a access token is suitable for authentication with a Minecraft server

validate(accessToken: String, clientToken: String): Promise<String>
Parameters
accessToken (String) required. The access token to be checked
clientToken (String) optional. If set, it should match the one used to obtain the access token
Returns
Promise<String>: returns an empty payload if successful
